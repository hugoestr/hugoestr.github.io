Interfaces are tests

Interfaces in static language like C# act like automatic interface tests. Use them.

Some dynamic language testing frameworks like rspec let you test that there is a certain interface implemented for an object. Years ago, as I was writing tests for the interface, I realized that this is something that static languages offer for free. In C# specifically interfaces act like a short-hand test to make sure that you are writing a specific number of methods for the object. So in a real sense the compiler is doing a unit test of the desired API.
